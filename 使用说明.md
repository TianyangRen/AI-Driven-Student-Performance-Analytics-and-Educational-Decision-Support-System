# 使用说明（本地启动指南）

AI 学生表现分析与教育决策支持系统 —— 三个子项目的安装与启动说明，覆盖
**Windows / macOS / Linux**。

---

## 1. 系统由三部分组成

| #   | 子项目       | 目录                     | 端口     | 作用                                                                      |
| --- | ------------ | ------------------------ | -------- | ------------------------------------------------------------------------- |
| ①   | **ML 服务**  | `ml-service/`            | **8000** | 机器学习/分析引擎（成绩预测、班级画像、预警、测评质量），独立 Django 服务 |
| ②   | **业务后端** | `web-platform/backend/`  | **8001** | Web 平台 API（账号、课程、成绩导入、分析、报告），通过 HTTP 调用 ①        |
| ③   | **前端**     | `web-platform/frontend/` | **5173** | React 管理界面，调用 ②                                                    |

**调用关系**：

```
浏览器 ──> 前端 :5173 ──(/api 代理)──> 业务后端 :8001 ──(HTTP)──> ML 服务 :8000
```

> ⚠️ 三个端口不要冲突。①②都是 Django，若都用默认 8000 会互相抢端口——本项目已固定
> 业务后端为 **8001**、ML 服务为 **8000**。

---

## 2. 前置要求

| 依赖           | 版本               | 说明                                  |
| -------------- | ------------------ | ------------------------------------- |
| **Python**     | 3.10+（建议 3.12） | 用于 ① 和 ② 两个后端                  |
| **Node.js**    | 18+（建议 20/22）  | 用于 ③ 前端                           |
| **Git**        | 任意               | 拉取代码                              |
| **uv**（可选） | 最新               | 更快的 Python 包管理，可替代 venv+pip |

> ①（ML 服务）和 ②（业务后端）**Django 版本不同，必须各用各的虚拟环境**，不要共用。

---

## 3. 首次安装（三个子项目）

先克隆代码并进入项目根目录：

```bash
git clone <仓库地址>
cd AI-Driven-Student-Performance-Analytics-and-Educational-Decision-Support-System
```

下面每个子项目分别建虚拟环境、装依赖。**每条命令块都给了 macOS/Linux 和 Windows 两种写法。**

---

### 3.1 ML 服务（`ml-service/`，端口 8000）

**创建虚拟环境 + 装依赖**

macOS / Linux：

```bash
cd ml-service
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Windows（PowerShell）：

```powershell
cd ml-service
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

> Windows 若报“禁止运行脚本”，先执行一次：
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`
> 或改用 CMD：`.venv\Scripts\activate.bat`

**训练模型**（预测类接口依赖它，只需一次；数据在已入库的 `real_data/*.csv`）：

```bash
python -m analytics.ml.train_real --save
```

**初始化 + 启动**：

```bash
python manage.py migrate          # 建表（消除迁移警告）
python manage.py runserver 8000
```

验证：浏览器打开 [http://127.0.0.1:8000/api/health/](http://127.0.0.1:8000/api/health/)，看到 `"loaded": true` 即模型就绪。

> 用 uv 的话：`uv venv` 建环境、`uv pip install -r requirements.txt` 装依赖、
> `uv run python manage.py runserver 8000` 启动（无需手动 activate）。

---

### 3.2 业务后端（`web-platform/backend/`，端口 8001）

**创建虚拟环境 + 装依赖**

macOS / Linux：

```bash
cd web-platform/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Windows（PowerShell）：

```powershell
cd web-platform\backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

**（可选）配置环境变量**：默认即可跑（SQLite + ML 指向 127.0.0.1:8000）。需要自定义时：

```bash
cp .env.example .env        # Windows: copy .env.example .env
# 按需修改 .env，如 ML_API_BASE_URL
```

**初始化数据库 + 灌入演示数据**（关键！见第 4 节说明）：

```bash
python manage.py migrate
python manage.py seed_all
```

**启动**：

```bash
python manage.py runserver 8001
```

验证：[http://127.0.0.1:8001/api/v1/health](http://127.0.0.1:8001/api/v1/health) 返回 `"status": "ok"`，其中 `ml.mode`
为 `real` 表示已连上 ML 服务（需 ① 已启动）。

---

### 3.3 前端（`web-platform/frontend/`，端口 5173）

```bash
cd web-platform/frontend
npm install
npm run dev
```

打开 [http://localhost:5173](http://localhost:5173)，用第 5 节的账号登录。

> 前端已配置把 `/api` 代理到 `http://localhost:8001`（业务后端）。若改了业务后端端口，
> 同步改 `web-platform/frontend/vite.config.js` 里的 `target`。

## 4. 为什么必须运行 `seed_all`？

**数据库文件 `db.sqlite3` 不入库（`.gitignore`）**，所以 `migrate` 只会建**空表**，
克隆下来是没有任何账号/课程/学生的。`seed_all` 负责重建完整可用的演示数据：

```bash
python manage.py seed_all
```

它会创建：

- 1 个超级用户 + 6 个教师账号
- 7 门课程、每门 2 个教学班
- 每个教学班 26 名学生（分层能力）+ 跨周成绩 + 出勤/参与
- 每个教学班的风险预测（需 ML 服务在线才是真实预测；不在线则自动降级为 mock）

**幂等**，可反复运行。常用参数：

```bash
python manage.py seed_all --students 30        # 每班 30 名学生
python manage.py seed_all --no-predict         # 只造数据不跑预测
python manage.py seed_all --password Mypass123 # 自定义演示账号密码
```

> 只想重刷学生成绩（不动账号/课程）时，用 `python manage.py seed_demo`。

## 5. 默认登录账号

`seed_all` 生成的账号（默认密码 `Passw0rd!`）：

| 用户名                  | 密码        | 角色   | 可见范围           |
| ----------------------- | ----------- | ------ | ------------------ |
| `superuser`             | `Passw0rd!` | 管理员 | 全部课程 / 教学班  |
| `TeacherA` … `TeacherF` | `Passw0rd!` | 教师   | 仅自己名下的教学班 |

> 教师登录只能看到自己的教学班，属于权限设计。想看全部数据用 `superuser`。

## 6. 日常启动（三个终端）

装好之后，每次开发开三个终端分别启动：

**终端 1 — ML 服务（:8000）**

```bash
cd ml-service
source .venv/bin/activate            # Windows: .venv\Scripts\Activate.ps1
python manage.py runserver 8000
```

**终端 2 — 业务后端（:8001）**

```bash
cd web-platform/backend
source .venv/bin/activate            # Windows: .venv\Scripts\Activate.ps1
python manage.py runserver 8001
```

**终端 3 — 前端（:5173）**

```bash
cd web-platform/frontend
npm run dev
```

然后访问 [http://localhost:5173](http://localhost:5173)。

## 7. 常见问题（FAQ）

**Q1：`ModuleNotFoundError: No module named 'django'`**
虚拟环境没激活，或没装依赖。确认命令提示符前有 `(.venv)`，再 `pip install -r requirements.txt`。

**Q2：`That port is already in use`（端口被占用）**
有旧进程占着端口。查杀：

- macOS/Linux：`lsof -tiTCP:8001 -sTCP:LISTEN | xargs kill`
- Windows：`netstat -ano | findstr :8001` 找到 PID，再 `taskkill /PID <PID> /F`

**Q3：页面报 “Failed to load” 或预测显示 mock**
ML 服务（:8000）没启动或模型没训练。先在 `ml-service/` 跑 `python -m analytics.ml.train_real --save` 再启动 ML 服务。

**Q4：`You have N unapplied migration(s)`**
没执行迁移。在对应后端目录运行 `python manage.py migrate`。

**Q5：页面数据是空的**
业务后端没灌数据。在 `web-platform/backend/` 运行 `python manage.py seed_all`。

**Q6：前端 401 / 一直跳登录页**
Token 失效或未登录，重新用第 5 节账号登录即可。

**Q7：ML 服务和业务后端能共用一个虚拟环境吗？**
不能。两者 Django 版本不同，必须各建各的 `.venv`。
